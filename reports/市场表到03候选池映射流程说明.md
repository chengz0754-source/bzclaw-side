# 03 与 60 候选池构建流程说明

## 当前口径

- P07 之后，`03_候选市场与候选品初筛池.csv` 不再是旧版“市场表直接映射后的中途人工判断表”。
- 当前 `03` 的正式角色是：
  - 中间候选样品池
  - 记录样品主键、来源关键词、去重组、近义合并组、上下游状态
  - 不在此阶段要求人工写最终判断

## 当前流转

1. 原始市场表 `.xlsx` 保留在 `runs/manual/10_market/`
2. STEP3 标准层输出 `31/32`
3. STEP4 标准层输出 `41/42`
4. P06 批跑层输出 `batch_queue_status.csv`
5. `python scripts/build_candidate_pool.py`
   - 读取批跑队列和 STEP3 / STEP4 结构化结果
   - 生成 runtime `03_候选市场与候选品初筛池.csv`
   - 生成 `60_候选样品池.csv` 与 `60_候选样品池.md`

## 旧 market-only 映射脚本的边界

- `scripts/map_market_report_to_candidate_pool.py` 仍可用于 market-only 清洗 / 辅助验证。
- 但它已经不是当前 `03` 角色的 authoritative builder。
- 当前 `03` 与 `60` 的 authoritative builder 是：
  - `scripts/build_candidate_pool.py`

## Fail-Closed 规则

- 如果 STEP2 仍 blocked，则允许只把真实 STEP3 / STEP4 已验证样品放进 `03` / `60`。
- 不允许为了“看起来完整”而伪造不存在的关键词链或样品链。
- 人工字段必须保持留空，等待白天人工阶段处理。
