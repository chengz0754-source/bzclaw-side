# 仓库业务文件流转与根目录卫生规则

## 推荐流转

```text
templates/selection_csv_cn_reference/
  -> inputs/selection_run_current/00~02
  -> runs/manual/10_market/*.xlsx
  -> outputs/selection_runs/<batch_id>/02_generated_outputs/30_31_32
  -> outputs/selection_runs/<batch_id>/02_generated_outputs/40_41_42
  -> scripts/build_candidate_pool.py
  -> outputs/selection_runs/<batch_id>/02_generated_outputs/03_候选市场与候选品初筛池.csv
  -> outputs/selection_runs/<batch_id>/02_generated_outputs/60_候选样品池.csv
  -> outputs/selection_runs/<batch_id>/02_generated_outputs/60_候选样品池.md
  -> scripts/build_sif_enrichment_daytime_pack.py
  -> outputs/selection_runs/<batch_id>/02_generated_outputs/50_51_52_53_61
  -> scripts/archive_selection_run_io.py
  -> outputs/selection_runs/<batch_id>/
```

## Nightly Acceptance Flow

夜跑验收推荐流转如下：

```text
inputs/selection_run_current/00~03
  -> scripts/run_selection_direction_batch.py
  -> scripts/build_candidate_pool.py
  -> scripts/collect_sif_detail_surface.py
  -> scripts/collect_sif_search_surface.py
  -> scripts/build_sif_enrichment_daytime_pack.py
  -> scripts/run_nightly_selection_acceptance.py
  -> outputs/selection_runs/<batch_id>/00_run_summary.md
  -> outputs/selection_runs/<batch_id>/01_consumed_inputs/
  -> outputs/selection_runs/<batch_id>/02_generated_outputs/
  -> outputs/selection_runs/<batch_id>/03_logs/
```

补充约束：

- `run_nightly_selection_acceptance.py` 是非破坏性的 acceptance runner，不会清空当前输入层。
- 只有 `archive_selection_run_io.py` 才是“把当前输入正式移走”的归档脚本。
- 若夜跑只得到 `HOLD` 或 `FAIL`，保留该批次作为证据，不回填、不伪造、不覆盖白天人工字段。

## 根目录卫生

- 仓库根目录不长期存放业务 CSV。
- 仓库根目录不长期存放原始市场 workbook、关键词导出或 benchmark 导出。
- 仓库根目录不长期存放临时业务说明文档。
- 模板层、当前输入层、raw workbook 层、运行输出层必须分层管理。

## 自动化治理

- `scripts/normalize_repo_business_files.py`
  - 扫描根目录业务文件污染
  - 默认只输出建议
  - 使用 `--apply` 时执行安全迁移
  - 迁移记录写入 `reports/仓库根目录业务文件迁移记录.md` 与 `.json`

## 当前分层原则

- 模板层：`templates/selection_csv_cn_reference/`
- 当前输入层：`inputs/selection_run_current/`
- raw workbook 层：`runs/manual/10_market/`
- 结构化运行输出层：`outputs/selection_runs/<batch_id>/02_generated_outputs/`
- 归档输出层：`outputs/selection_runs/<batch_id>/`

## 当前使用顺序

1. 从模板层重置当前输入层 `00~02`。
2. 人工确认 `00/01/02`。
3. 将 SellerSprite market workbook 落到 `runs/manual/10_market/`。
4. 运行 STEP3 / STEP4 标准脚本，生成 `30/31/32` 与 `40/41/42`。
5. 运行 `python scripts/build_candidate_pool.py` 生成 runtime `03` 与 `60`。
6. 运行 SIF 相关脚本生成 `50/51/52/53`，并仅在前五阶段均 `PASS` 时产出 `61`。
7. 夜间验收时使用 `python scripts/run_nightly_selection_acceptance.py --batch-id <batch_id>`。
8. 白天人工确认后，如需正式收口，再执行 `archive_selection_run_io.py`。

## 特别说明

- `scripts/map_market_report_to_candidate_pool.py` 不再是当前 `03` 的 authoritative writer。
- 当前 `03_候选市场与候选品初筛池.csv` 是中间候选样品池，不要求中途人工写最终判断。
- `60_候选样品池.csv` 是白天可读候选池投影，`合规 / 改良点 / 最终解释 / 利润核价 / 最终 Go/No-Go` 必须保持留空。
